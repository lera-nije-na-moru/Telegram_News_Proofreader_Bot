# Telegram_News_Proofreader_Bot
Автоматический бот для исправления технических опечаток в новостном Telegram-канале. Pet-project, созданный для автоматизации рутинной модерации:
- исправление опечаток в подписи канала,
- нормализация "ё" → "е",
- корректировка username сообщества,
- пересборка Telegram-альбомов без потери форматирования.

Проект написан на aiogram (async).

##  Возможности
- Фильтрация по сигнатуре поста
Бот обрабатывает только сообщения, содержащие стандартный "подвал":
---
Подпишись @username ...
Дежурный админ @username ...
размещаем анонимно

- Если подписи нет — сообщение игнорируется.
Это защищает:
- рекламные посты,
- партнёрские публикации,
- нестандартные сообщения.

## Автоматические исправления
Реализовано:
- ё → е
- Ё → Е
- @..._vao → @..._svao

Логика легко расширяется через process_text().

## Сохранение Telegram-разметки
Бот корректно работает с:
- entities
- caption_entities
- ссылками
- упоминаниями
- форматированием

Текст изменяется без потери разметки.

### Корректная обработка альбомов (media_group)
Telegram не позволяет редактировать альбом целиком. Бо ждёт, пока соберётся весь media_group, исправляет подпись первого элемента, пересобирает альбом через send_media_group и удаляет оригинальные сообщения. По времени занимает меньше секунды.

## Защита от обработки старых сообщений
Бот работает только с постами, отправленными после запуска:
if msg_date < BOT_START_TIME:
    return
Это исключает случайную переработку истории канала.

## Архитектура
- SIGNATURE_PATTERN — regex-фильтр подписи
- normalize_yo() — нормализация ё/е
- normalize_text_with_entities() — безопасная правка текста
- media_groups — временное хранилище альбомов
- async polling через aiogram

## Запуск
- Установка
- pip install aiogram
- Конфигурация
- API_TOKEN = "YOUR_TOKEN"
- Старт
- python bot.py
- Production Safety

Проект изначально разрабатывался с учётом продакшен-рисков:
- обработка только сообщений с обязательной сигнатурой
- игнорирование рекламных постов
- работа только с новыми сообщениями
- пересборка вместо редактирования
- сохранение entities
- защита от частичной обработки альбомов

Дополнительно можно реализовать:
- dry-run режим
- логирование в файл
- white-list чатов
- тестовый режим для staging-канала
